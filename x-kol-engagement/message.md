## Engagement Principles


### The 80/20 Rule

- **80% value-add** — insight, analysis, thoughtful commentary, questions
- **20% ZION-relevant** — only when there's a genuine connection to what ZION is building

### The "Would I Say This at a Conference?" Test

Before drafting any reply, ask: *Would I say this face-to-face at a crypto conference to this person?*
- If yes → proceed
- If no → rethink or skip


### Engagement Tiers

| KOL Priority | Engagement Style |
|-------------|-----------------|
| **High** | Substantive technical replies. Add original insight. Reference shared context if any. Be memorable. |
| **Medium** | Thoughtful one-liner or brief take. Add perspective. Like + reply combo. |
| **Low** | Like only, or brief supportive comment. Don't over-invest. |

## Reply Templates


These are structural patterns, NOT copy-paste templates. Every reply must be original and contextual.


### Pattern 1 — Technical Insight Reply

**Use when:** KOL posts about a technical topic you have genuine knowledge on.

```
Structure:
[Acknowledge their point] + [Add your technical angle] + [Optional: connect to broader thesis]
```

**Example:**
> KOL: "Account abstraction is finally making wallets usable for normal people."
>
> Reply: "AA is the unlock, but the next step is agent-native wallets — accounts that aren't operated by humans at all but by autonomous agents executing strategies 24/7. The UX problem inverts: it's not about making wallets simpler for people, it's about making them programmable for agents."


### Pattern 2 — Builder Empathy Reply

**Use when:** KOL shares a shipping milestone, launch, or building update.

```
Structure:
[Genuine acknowledgment of the work] + [Specific detail you noticed] + [Optional: relate to your own building experience]
```

**Example:**
> KOL: "After 6 months, we finally shipped our cross-chain bridge to mainnet."
>
> Reply: "6 months of bridge security audits — that's the part people don't see. The fact that you didn't rush to mainnet says a lot. Congrats on doing it right."


### Pattern 3 — Contrarian / Nuanced Take

**Use when:** KOL posts a take you partially agree with but have a different angle on.

```
Structure:
[Agree with part] + [Introduce nuance or alternative framing] + [Support with reasoning]
```

**Example:**
> KOL: "AI agents will replace all crypto trading bots."
>
> Reply: "Replace is strong — more like subsume. The interesting shift isn't agents vs bots, it's that agents can compose across protocols autonomously. A trading bot executes one strategy. An agent can rebalance, bridge, stake, and hedge in a single reasoning loop."


### Pattern 4 — Question / Curiosity

**Use when:** KOL posts something thought-provoking and you want to engage deeper.

```
Structure:
[Reference their specific point] + [Ask a genuine question that shows depth]
```

**Example:**
> KOL: "We need better identity primitives for DeFi."
>
> Reply: "Curious what you think about agent identity specifically — not just 'who is this human' but 'who is this autonomous entity and what's its track record?' That's a very different design space."


### Pattern 5 — Signal Boost with Commentary

**Use when:** KOL shares something important that deserves amplification with your added context.

```
Structure:
[Your take / why this matters] + quote tweet the original
```

**Example:**
> Quote tweet: "This is undersold. On-chain reputation for agents isn't just nice-to-have — it's the trust layer that makes agent-to-agent commerce possible. No reputation = no delegation = no autonomy."

## ZION Reference Guidelines


### When to Mention ZION

✅ **Do mention ZION when:**
- The conversation is directly about autonomous agents, agent identity, or agent networks
- Someone asks "what are you building?" or "tell me more"
- You're drawing a genuine parallel between the discussion topic and ZION's architecture
- You're responding to a direct question about your project

### How to Mention ZION

- **Natural, not forced:** "We're tackling this exact problem at ZION — agent identity on-chain"
- **Specific, not vague:** Reference a concrete aspect (identity layer, heartbeat protocol, agent registry)
- **Brief, not pitch-y:** One sentence max. Let them ask for more.

### When NOT to Mention ZION

❌ **Do NOT mention ZION when:**
- The topic has no connection to what ZION does
- It would feel forced or shoehorned
- You'd be hijacking someone else's announcement thread
- The conversation is about a sensitive topic (hacks, losses, regulatory action)
- You've already mentioned ZION in a recent reply to the same person

## Anti-Patterns — NEVER Do These


### ❌ The Generic GM
```
BAD:  "gm! great thread 🔥"
BAD:  "this is so based"
BAD:  "wagmi 🚀"
```
These add zero value and make you look like a bot.


### ❌ The Cold Shill
```
BAD:  "Have you heard of ZION? We're building an autonomous agent network. Check it out at testnet.zion.space"
BAD:  "This is exactly what ZION solves! Come see our demo."
```
Unsolicited project promotion in someone else's thread is spam.


### ❌ The Reply Guy
```
BAD:  Replying to every single tweet from a KOL
BAD:  Replying within seconds of a post (looks automated)
BAD:  Multiple replies in the same thread
```
Quality over quantity. 1 great reply > 10 mid replies.


### ❌ The Agreement Bot
```
BAD:  "Totally agree!"
BAD:  "This 👆"
BAD:  "100% correct"
```
Empty agreement adds nothing. If you agree, say WHY.


### ❌ The Over-Explainer
```
BAD:  [500-word essay in a reply]
```
X is a short-form platform. Keep replies under 280 characters when possible. If you need more space, quote tweet instead.


### ❌ The Engagement Farmer
```
BAD:  "What do you think about [unrelated topic]?" to bait engagement
BAD:  Tagging other people to draw attention
BAD:  "Follow us for more insights on..."
```


### ❌ Sensitive Topic Engagement
```
BAD:  Engaging with tweets about hacks, exploits, or losses to promote ZION
BAD:  Engaging with political content
BAD:  Engaging with personal drama or disputes
BAD:  Giving financial advice ("you should buy/sell X")
```

## Reply Length Guidelines


| Context | Target Length |
|---------|-------------|
| Like + brief support | 1 line, < 100 chars |
| Standard reply | 1–3 sentences, < 280 chars |
| Technical insight | 2–4 sentences, < 500 chars |
| Quote tweet commentary | 1–3 sentences, < 280 chars |


### Formatting Tips

- Use line breaks for readability in longer replies
- Avoid excessive emojis (1–2 max per reply)
- No hashtags in replies (looks spammy)
- Use quotes or dashes for emphasis, not ALL CAPS
- Thread replies (multi-tweet) only for your OWN original content, never in someone else's thread
